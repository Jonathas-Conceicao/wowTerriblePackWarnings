# Requirements: TerriblePackWarnings

**Defined:** 2026-03-17
**Core Value:** When a player imports a route and pulls, they see accurate, timed ability warnings via custom spell icon display with per-mob detection.

## v0.1.0 Requirements

Requirements for Configuration and Skill Data milestone. Each maps to roadmap phases.

### Ability Data

- [x] **DATA-13**: All 9 Midnight S1 dungeons have ability data files with spellIDs extracted from MDT source
- [x] **DATA-14**: Mobs default to WARRIOR class when imported from MDT (MDT lacks class data)
- [x] **DATA-15**: Spell names and icons resolved dynamically via C_Spell.GetSpellInfo at addon load

### Configuration UI

- [ ] **CFG-01**: Config window with left panel showing dungeon list expandable to deduplicated mobs
- [ ] **CFG-02**: Right panel shows selected mob's skills with per-skill settings
- [x] **CFG-03**: User can toggle tracking on/off for each skill via checkbox
- [x] **CFG-04**: User can set custom label per skill (current label as default, empty allowed)
- [ ] **CFG-05**: User can see WoW spell tooltip when hovering a skill in the config window

### Alerts

- [x] **ALERT-01**: Per-skill sound alert dropdown with WoW built-in sounds (CDM-style, categorized, preview on select)
- [x] **ALERT-02**: Per-skill editable TTS text field (current text as default)
- [x] **ALERT-03**: Alert type is sound OR TTS per skill (mutually exclusive)

### Highlighting

- [ ] **HILITE-01**: Untimed skills highlight when any mob of same class is casting (via UnitCastingInfo nameplate polling)
- [x] **HILITE-02**: Timed skills highlight 5 seconds before cast with configured alert

### Route Management

- [x] **ROUTE-01**: Each dungeon stores its own imported route independently in SavedVariables
- [x] **ROUTE-02**: TPW window has dungeon selector to switch active dungeon view
- [x] **ROUTE-03**: Active dungeon auto-switches on zone-in via GetInstanceInfo
- [x] **ROUTE-04**: Pull rows show mob count per type (e.g. "3x Spellguard")

### Commands and Search

- [x] **CMD-01**: `/tpw` opens config window; `/tpw route` opens route window; commands case-insensitive
- [x] **CMD-02**: `/tpw help` shows complete list of available commands with descriptions
- [x] **CMD-03**: Remove config button from route window; add route button to config window
- [x] **SEARCH-01**: Config window has search box that filters dungeon/mob tree by mob name and skill name
- [x] **SEARCH-02**: Selecting a mob while search active shows only matching skills; closing window resets search
- [x] **UIPOL-01**: Reposition Reset All button to work with new search box layout
- [x] **UIPOL-02**: Update remaining button layout in route window after config button removal

### Profiles and Skill Config Rework

- [x] **PROF-01**: All skills default to unchecked, untimed, empty timer fields — no pre-enabled skills
- [x] **PROF-02**: Per-skill timed/untimed toggle with first_cast and cooldown number inputs (grayed out when untimed)
- [x] **PROF-03**: Profile system — account-wide profiles stored in SavedVariables, default profile is blank
- [x] **PROF-04**: Profile selector dropdown in config window with "New Profile" option (creates "Profile 1", "Profile 2", etc.)
- [x] **PROF-05**: Delete profile button (disabled for default profile)
- [x] **PROF-06**: Import/export profile buttons (serialized string for sharing)
- [x] **PROF-07**: Per-skill sound alert checkbox — untimed plays on class cast detection, timed plays at 5s pre-warning

## Out of Scope

| Feature | Reason |
|---------|--------|
| Combat log event parsing | Blocked by Midnight API restrictions (CLEU disabled) |
| Auto-detection of current pack without route | No reliable API |
| Route editing within TPW | Editing stays in MDT/Keystone.guru |
| Sound AND TTS simultaneously | User chose mutually exclusive alert types |
| Sound throttle | Deferred — evaluate after testing multi-mob scenarios |
