**2. Add search filter function** (before BuildConfigFrame, after BuildLeftPanel):

Create `ApplySearchFilter(text)`:
- If text is empty or nil, clear `currentMatchedMobs` and `currentMatchedSpells`, set `currentSearchText = ""`, rebuild all nodes as visible (set `node.visible = true` for all), call `RebuildLayout()`, and if `selectedNpcID` then re-call `PopulateRightPanel(selectedNpcID)`. Return.
- Lowercase the text: `local filter = text:lower()`
- Set `currentSearchText = filter`
- Clear `currentMatchedMobs` and `currentMatchedSpells`
- Iterate the full dungeon index (call `BuildDungeonIndex()`) and for each mob:
  - Check if mob name matches: `(mob.name or ""):lower():find(filter, 1, true)`
  - Check if any ability name matches by iterating `ns.AbilityDB[mob.npcID].abilities` and calling `GetSpellNameSafe(ability):lower():find(filter, 1, true)`
  - If mob name matches: `currentMatchedMobs[mob.npcID] = true` (all spells shown)
  - If only specific spells match: `currentMatchedMobs[mob.npcID] = true` and `currentMatchedSpells[mob.npcID] = { [matchedSpellID] = true, ... }`
  - If mob name matches, do NOT populate `currentMatchedSpells` for that mob (show all abilities)
- Mark tree nodes: for each node in `nodes`, set `node.visible = false`. Then for each mob match, find the node with matching `node.dungeonIdx` and set `node.visible = true`.
- Auto-expand visible dungeon nodes: for each node where `node.visible == true`, set `node.expanded = true` and update header text to `"[-] " .. node.dungeonName`.
- Collapse non-visible nodes: for each node where `node.visible == false`, set `node.expanded = false` and update header text to `"[+] " .. node.dungeonName`.
- Also hide mob rows within visible content frames that don't match. Each mob row in the content frame needs a way to be shown/hidden. Store mob rows on the node: `node.mobRows = { { frame = mobRow, npcID = npcID }, ... }`. During filtering, iterate `node.mobRows` and show/hide based on `currentMatchedMobs[row.npcID]`. Recompute `node.contentHeight` based on visible mob count.
- Call `RebuildLayout()`.
- If `selectedNpcID` is in `currentMatchedMobs`, re-call `PopulateRightPanel(selectedNpcID, currentMatchedSpells[selectedNpcID])`. Otherwise clear selection: `selectedNpcID = nil`, set right panel header to "Select a mob to view abilities."
