**Pipeline.lua — MergeSkillConfig (line 32-34 area, the "no override" branch):**

Currently when `cfg` is nil (no user override), MergeSkillConfig returns the ability unchanged. Add a check for `ability.defaultEnabled == false` BEFORE the existing return. The new logic:

```lua
if not cfg then
    -- No user override exists. Check if ability defaults to disabled.
    if ability.defaultEnabled == false then return nil end
    -- Return ability unchanged (copy to avoid mutating AbilityDB)
    return {
        name       = ability.name,
        spellID    = ability.spellID,
        mobClass   = mobClass,
        first_cast = ability.first_cast,
        cooldown   = ability.cooldown,
        label      = ability.label,
        ttsMessage = ability.ttsMessage,
    }
end
```

This means: if no user has ever toggled this ability AND it has `defaultEnabled = false`, it is omitted from packs. Once a user explicitly enables it via the checkbox (writing `cfg.enabled = nil`), the cfg branch takes over and the ability is included.

Also update the existing `cfg.enabled == false` branch comment to clarify:
```lua
if cfg.enabled == false then return nil end  -- user explicitly disabled
```

**ConfigFrame.lua — checkbox initial state (around line 337):**

Currently the checkbox is unchecked only when `cfg and cfg.enabled == false`. For abilities with `defaultEnabled = false` and no user override (cfg is nil), the checkbox should also be unchecked.

Find the block:
```lua

    checkBtn:SetChecked(false)
else
    checkBtn:SetChecked(true)
end
```

Replace with:
```lua
if cfg and cfg.enabled == false then
    checkBtn:SetChecked(false)
elseif not cfg and ability.defaultEnabled == false then
    checkBtn:SetChecked(false)
else
    checkBtn:SetChecked(true)
end
```

The `ability` variable is already in scope at that point (it comes from the skill iteration loop). Verify by reading the surrounding code that `ability` is the current AbilityDB entry being rendered.

**ConfigFrame.lua — checkbox OnClick handler:**

When the user checks a `defaultEnabled = false` ability for the first time, we need to write `cfg.enabled = nil` (which means "user has explicitly enabled"). The current OnClick already does this:
```lua
if self:GetChecked() then
    ns.db.skillConfig[npcID_cb][spellID_cb].enabled = nil
```
This is correct — writing `enabled = nil` creates the cfg entry, so MergeSkillConfig's `if not cfg then` branch no longer fires, and the `cfg.enabled == false` check is false (because enabled is nil, not false). No change needed to OnClick.
