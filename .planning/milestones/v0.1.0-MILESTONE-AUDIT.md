## Integration Issues

### Issue 1: combatMode forced to "disable" on non-S1 zone (Medium)
CombatWatcher.Reset() unconditionally sets `ns.db.combatMode = "disable"` when leaving S1 dungeons. This is by design (user requested auto-disable on zone-out), but means the user must re-enable Auto/Manual every time they enter a dungeon. This is documented behavior in the README.

### Issue 2: timed/timer/label changes don't rebuild packs immediately (Low-Medium)
Changes rebuild on config close (OnHide calls RestoreAllFromSaved). Not a blocker — config close is the natural workflow boundary.

### Issue 3: MergeSkillConfig bypasses Profile.GetSkillConfig (Low)
Direct access to ns.db.profiles instead of calling Profile.GetSkillConfig(). Functionally equivalent but couples Pipeline to profile internals.

### Issue 4: RestoreAllFromSaved resets pack position (Low)
SelectDungeon called on every rebuild, resetting currentPackIndex to 1. Mid-run config changes lose pack position.

### Issue 5: ZONE_DUNGEON_MAP and DUNGEON_IDX_MAP are independent (Low)
Both maintained separately. Currently consistent but fragile if one is updated without the other.

## Recommendation

**Status: tech_debt** — All requirements met, no critical blockers. Issues are quality-of-life improvements suitable for a future minor patch, not blockers for v0.1.0 release.
