<task type="auto">
  <name>Task 2: Create display abstraction module with 3-tier fallback</name>
  <files>Display/BossWarnings.lua</files>
  <action>
Create `Display/BossWarnings.lua` implementing the display abstraction per locked decision (addon is data provider only, no custom frames in v1 except the guaranteed fallback).

The module exposes:
- `ns.BossWarnings.Show(text, duration)` — Display a text warning (for pre-warnings and cast alerts)
- `ns.BossWarnings.ShowTimer(id, text, duration, spellID)` — Start a countdown timer bar/event (for the Encounter Timeline and DBM paths)
- `ns.BossWarnings.CancelTimer(id)` — Cancel a specific timer by ID
- `ns.BossWarnings.CancelAllTimers()` — Cancel all active timers
- `ns.BossWarnings.GetAdapter()` — Returns name of active adapter (for debugging)

**Adapter detection** (runs once on first call, cached):
1. Check `C_EncounterTimeline and C_EncounterTimeline.IsFeatureEnabled and C_EncounterTimeline.IsFeatureEnabled()` — if true, use Encounter Timeline adapter
2. Check `DBT` global exists — if true, use DBM timer bar adapter
3. Otherwise, use RaidNotice_AddMessage fallback

**Encounter Timeline adapter:**
- `ShowTimer(id, text, duration, spellID)` calls `C_EncounterTimeline.AddScriptEvent()` with appropriate eventInfo table. Store returned eventID keyed by our `id` for cancellation.
- `CancelTimer(id)` calls `C_EncounterTimeline.CancelScriptEvent(storedEventID)`
- `CancelAllTimers()` calls `C_EncounterTimeline.CancelAllScriptEvents()`
- `Show(text, duration)` uses `RaidNotice_AddMessage` for text flash (Encounter Timeline handles bars, text flash is supplementary)

**DBM adapter:**
- `ShowTimer(id, text, duration, spellID)` calls `DBT:CreateBar(duration, "TPW_" .. id, spellID)` — prefix ID with "TPW_" to avoid collisions
- `CancelTimer(id)` calls `DBT:CancelBar("TPW_" .. id)`
- `CancelAllTimers()` calls `DBT:CancelAllBars()` — NOTE: this cancels ALL DBM bars, which is aggressive. Instead, track our bar IDs in a table and cancel individually.
- `Show(text, duration)` uses `RaidNotice_AddMessage` for text flash

**RaidNotice fallback adapter:**
- `ShowTimer(id, text, duration, spellID)` — No bar support; call `Show(text .. " (" .. duration .. "s)", duration)` as a text approximation
- `CancelTimer(id)` — No-op (text flash cannot be cancelled)
- `CancelAllTimers()` — No-op
- `Show(text, duration)` calls `RaidNotice_AddMessage(RaidBossEmoteFrame, text, ChatTypeInfo["RAID_WARNING"], duration or 5)`

**Important implementation details:**
- Use `local addonName, ns = ...` namespace pattern
- Initialize `ns.BossWarnings = {}`
- Adapter detection should be lazy (first call) not at load time, because C_EncounterTimeline may not be available until the player is in a dungeon
- For DBM adapter, do NOT call `DBT:CancelAllBars()` — track our own bar IDs in a local table `activeBarIDs` and cancel each individually
- Print adapter name to chat on first detection: `print("|cff00ccffTPW|r display: " .. adapterName)`
