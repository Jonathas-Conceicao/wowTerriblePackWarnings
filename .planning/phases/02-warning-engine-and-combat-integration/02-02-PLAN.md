  <action>
Create `Engine/Scheduler.lua` implementing the timer scheduling engine.

**Namespace:** `ns.Scheduler = {}`

**State:**
- `local activeTimers = {}` — all C_Timer handles for bulk cancel
- `local combatActive = {false}` — single-element table (mutable ref for closures, per research anti-pattern guidance)
- `local timerCounter = 0` — incrementing ID for display timer IDs

**Public API:**

`ns.Scheduler:Start(dungeonKey, packIndex)`
- Look up `ns.PackDatabase[dungeonKey][packIndex]`
- If nil, print error and return
- Set `combatActive[1] = true`
- For each mob in pack.mobs, for each ability in mob.abilities:
  - Call `scheduleAbility(ability)` (local function)
- Print pack name to chat: `"|cff00ccffTPW|r Started: " .. pack.displayName`

`ns.Scheduler:Stop()`
- Set `combatActive[1] = false`
- Cancel all timers in `activeTimers` using `:Cancel()` (check `:IsCancelled()` first)
- `wipe(activeTimers)`
- Call `ns.BossWarnings.CancelAllTimers()`
- Reset `timerCounter = 0`

**Local function `scheduleAbility(ability)`:**

Implements the recursive timer chain pattern from research:

1. Calculate `preWarnOffset = ability.first_cast - 5` (clamp to 0 if negative)
2. If `preWarnOffset > 0`, schedule pre-warning timer:
   ```lua
   local preHandle = C_Timer.NewTimer(preWarnOffset, function()
       if not combatActive[1] then return end
       ns.BossWarnings.Show(ability.name .. " in 5 sec", 4)
   end)
   table.insert(activeTimers, preHandle)
   ```
3. Schedule cast timer:
   ```lua
   timerCounter = timerCounter + 1
   local timerId = "ability_" .. timerCounter
   local castHandle = C_Timer.NewTimer(ability.first_cast, function()
       if not combatActive[1] then return end
       ns.BossWarnings.Show(ability.name, 3)
       -- Reschedule for next repeat using cooldown as the new first_cast
       scheduleAbility({
           name = ability.name,
           spellID = ability.spellID,
           first_cast = ability.cooldown,
           cooldown = ability.cooldown,
       })
   end)
   table.insert(activeTimers, castHandle)
   ```
4. Also schedule a ShowTimer call for the countdown bar (if adapter supports it):
   ```lua
   timerCounter = timerCounter + 1
   local barId = "bar_" .. timerCounter
   ns.BossWarnings.ShowTimer(barId, ability.name, ability.first_cast, ability.spellID)
   ```

**Important implementation details:**
- Use `local addonName, ns = ...` namespace pattern
- `combatActive` MUST be a single-element table, not a boolean. Closures in Lua capture variable references, but boolean reassignment creates a new value. The table reference stays the same so closures see the update via `combatActive[1]`.
- Every `C_Timer.NewTimer` handle MUST go into `activeTimers` — no exceptions. Ghost warnings are the #1 pitfall.
- Do NOT use `C_Timer.After` (no cancellation API).
- The recursive `scheduleAbility` call in the cast callback creates the repeating cycle. Each recursion schedules fresh timers with `ability.cooldown` as the delay.

**Engine/CombatWatcher.lua** — Create the combat lifecycle manager.

**Namespace:** `ns.CombatWatcher = {}`

**State:**
- `local selectedDungeon = nil` — dungeon key string (e.g., "windrunner_spire")
- `local currentPackIndex = nil` — 1-based index into the dungeon's pack array
- `local state = "idle"` — one of: "idle", "ready", "active", "end"

State machine:
- `idle`: No dungeon selected. PLAYER_REGEN_DISABLED does nothing.
- `ready`: Dungeon selected, waiting for combat. PLAYER_REGEN_DISABLED starts timers.
- `active`: Timers running. PLAYER_REGEN_ENABLED stops timers and advances.
- `end`: All packs exhausted. PLAYER_REGEN_DISABLED does nothing.

**Public API:**

`ns.CombatWatcher:SelectDungeon(dungeonKey)`
- Validate `ns.PackDatabase[dungeonKey]` exists and has entries
- Set `selectedDungeon = dungeonKey`
- Set `currentPackIndex = 1`
- Set `state = "ready"`
- Print: `"|cff00ccffTPW|r Selected: " .. dungeonKey .. " (" .. #ns.PackDatabase[dungeonKey] .. " packs)"`

`ns.CombatWatcher:ManualStart(packIndex)`
- If `selectedDungeon` is nil, print error and return
- If `packIndex` provided, set `currentPackIndex = packIndex`
- Call `ns.Scheduler:Start(selectedDungeon, currentPackIndex)`
- Set `state = "active"`

`ns.CombatWatcher:OnCombatStart()`
- If `state ~= "ready"` then return (guard against no-selection and end-state per pitfall #3)
- Call `ns.Scheduler:Start(selectedDungeon, currentPackIndex)`
- Set `state = "active"`

`ns.CombatWatcher:OnCombatEnd()`
- If `state ~= "active"` then return
- Call `ns.Scheduler:Stop()`
- Auto-advance: `currentPackIndex = currentPackIndex + 1`
- Check if `currentPackIndex > #ns.PackDatabase[selectedDungeon]`:
  - Yes: Set `state = "end"`, print `"|cff00ccffTPW|r All packs completed."`
  - No: Set `state = "ready"`, print `"|cff00ccffTPW|r Next: " .. ns.PackDatabase[selectedDungeon][currentPackIndex].displayName`

`ns.CombatWatcher:Reset()`
- Call `ns.Scheduler:Stop()`
- If `selectedDungeon` and `ns.PackDatabase[selectedDungeon]` exist:
  - Set `currentPackIndex = 1`
  - Set `state = "ready"`
  - Print `"|cff00ccffTPW|r Reset to pack 1"`
- Else:
  - Set `state = "idle"`

`ns.CombatWatcher:GetState()` — Returns `state, selectedDungeon, currentPackIndex` (for debugging and Phase 3 UI)

**Core.lua** — Update the event frame to wire combat events to CombatWatcher.

Changes to make:
1. **Remove** `self:UnregisterEvent("PLAYER_ENTERING_WORLD")` from the PLAYER_ENTERING_WORLD handler (pitfall #2 — must stay registered for zone-change resets)
2. **Add** `PLAYER_REGEN_DISABLED` and `PLAYER_REGEN_ENABLED` event registration
3. **Add** event handlers that call CombatWatcher methods
4. **Keep** the existing ADDON_LOADED handler unchanged

The updated OnEvent handler should be:
```lua
frame:SetScript("OnEvent", function(self, event, ...)
    if event == "ADDON_LOADED" then
        local name = ...
        if name ~= addonName then return end
        if not TerriblePackWarningsDB then
            TerriblePackWarningsDB = {}
        end
        ns.db = TerriblePackWarningsDB
        print("|cff00ccffTerriblePackWarnings|r loaded. Type |cff00ff00/tpw|r to configure.")
        self:UnregisterEvent("ADDON_LOADED")

    elseif event == "PLAYER_REGEN_DISABLED" then
        ns.CombatWatcher:OnCombatStart()

    elseif event == "PLAYER_REGEN_ENABLED" then
        ns.CombatWatcher:OnCombatEnd()

    elseif event == "PLAYER_ENTERING_WORLD" then
        ns.CombatWatcher:Reset()
    end
end)
```

Also register the new events:
```lua
frame:RegisterEvent("PLAYER_REGEN_DISABLED")
frame:RegisterEvent("PLAYER_REGEN_ENABLED")
```

Do NOT unregister PLAYER_ENTERING_WORLD — it must fire on every zone change.
