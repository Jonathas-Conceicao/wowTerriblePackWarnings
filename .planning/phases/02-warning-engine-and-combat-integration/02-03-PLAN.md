In Engine/Scheduler.lua, wrap the BossWarnings.CancelAllTimers() call at line 95 in pcall:

```lua
-- In Scheduler:Stop(), replace line 95:
--   ns.BossWarnings.CancelAllTimers()
-- With:
local ok, err = pcall(ns.BossWarnings.CancelAllTimers)
if not ok then
    print("|cff00ccffTPW|r Warning: CancelAllTimers error: " .. tostring(err))
end
```

This ensures Stop() always completes even if the display adapter throws (e.g., C_EncounterTimeline.CancelAllScriptEvents unavailable outside instances).

In Engine/CombatWatcher.lua, update OnCombatEnd() to set state BEFORE calling Stop(), so even if Stop() somehow fails, state is no longer "active":

```lua
function CombatWatcher:OnCombatEnd()
    if state ~= "active" then return end

    -- Transition state BEFORE Stop() to prevent re-triggering on error
    local dungeon = ns.PackDatabase[selectedDungeon]
    local nextIndex = currentPackIndex + 1

    if nextIndex > #dungeon then
        state = "end"
        currentPackIndex = nextIndex
        ns.Scheduler:Stop()
        print("|cff00ccffTPW|r All packs completed.")
    else
        state = "ready"
        currentPackIndex = nextIndex
        ns.Scheduler:Stop()
        print("|cff00ccffTPW|r Next: " .. dungeon[currentPackIndex].displayName)
    end
end
```

Key change: state and currentPackIndex are updated BEFORE Stop() runs. If Stop() errors, state is already "ready" or "end", preventing OnCombatStart from re-triggering on the same pack.
