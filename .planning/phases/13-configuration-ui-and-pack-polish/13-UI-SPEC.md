# Phase 13 — UI Design Contract

> Visual and interaction contract for WoW addon UI phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This is a World of Warcraft addon (Lua + FrameXML). All UI is built with WoW's native frame system. There are no CSS, HTML, React, or web technologies.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | WoW native FrameXML (Lua API only) |
| Preset | Not applicable — no shadcn, no Tailwind |
| Component library | WoW built-in frame templates (BasicFrameTemplateWithInset, GameMenuButtonTemplate, UIPanelScrollFrameTemplate) |
| Icon library | WoW spell icons via C_Spell.GetSpellTexture(spellID); class icons via Interface\\Icons\\ClassIcon_* |
| Font | FRIZQT__.TTF (WoW standard) at sizes 10, 12, 14; font objects: GameFontNormal, GameFontHighlight, GameFontNormalSmall, ChatFontNormal |

**shadcn gate:** Not applicable. This is not a React/Next.js/Vite project. WoW addon UI uses WoW FrameXML only.

---

## Spacing Scale

WoW uses pixel-based positioning via SetPoint/SetSize. Values follow a 4-point scale matching existing PackFrame.lua constants.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 2px | Portrait-to-portrait gap (matches `row.portraits` 2px gap in PackFrame.lua) |
| sm | 4px | Pull number left padding, icon inner padding, label gap below icon |
| md | 8px | Button-to-button gap (matches importBtn/clearBtn 8px gap), widget-to-widget within skill row |
| lg | 12px | ScrollFrame TOPLEFT inset from frame edge (matches PackFrame.lua anchor pattern) |
| xl | 16px | Padding between skill row sections within right panel |
| 2xl | 24px | Vertical gap between dungeon header and first mob row |
| 3xl | 34px | ScrollFrame BOTTOMRIGHT inset (matches UIPanelScrollFrameTemplate scrollbar width allowance) |

Exceptions:
- Config window title bar: 28px below TOPLEFT for header label (matches existing `header:SetPoint` pattern)
- Config window footer button zone: 35px from BOTTOMRIGHT (matches PackFrame.lua footer zone)
- NPC portrait size: 22px × 22px (matches `PORTRAIT_SIZE = 22` in PackFrame.lua)
- Count label font size: 10px OUTLINE (matches RESEARCH.md Pattern 5 spec)

## Typography

All typography uses WoW font objects and FRIZQT__.TTF. WoW has no CSS line-height concept — font objects handle vertical spacing implicitly.

| Role | Font Object / Literal | Size | Style | Usage |
|------|-----------------------|------|-------|-------|
| Window title | TitleText (from BasicFrameTemplateWithInset) | ~14px (template default) | Normal | ConfigFrame and PackFrame window titles |
| Section header | GameFontNormal | 12px | Normal | Dungeon collapsible headers in left panel; right panel mob name + class header |
| Body / mob name | GameFontNormal | 12px | Normal | Mob name in tree rows; pull number in PackFrame |
| Small label | GameFontNormalSmall | 10px | Normal | Timing info row ("First cast: 50s, Cooldown: 50s") — read-only, de-emphasized |
| Icon label | FRIZQT__.TTF, 14px, OUTLINE | 14px | Outline | Spell icon label below icon (matches IconDisplay.lua lbl pattern) |
| Count overlay | FRIZQT__.TTF, 10px, OUTLINE | 10px | Outline | "x3" portrait count overlay in PackFrame pull rows |
| EditBox input | ChatFontNormal | ~12px | Normal | Label EditBox, TTS text EditBox, import EditBox (matches importPopup pattern) |
| Disabled text | GameFontDisable | 12px | Normal | TTS EditBox text when grayed out (sound selected in dropdown) |

**Weight:** WoW font objects control weight implicitly. GameFontNormal = regular weight. No bold weights used except in Blizzard template title bars.

## Color

WoW colors are RGBA tuples set via SetColorTexture or SetTextColor. No hex — all values are 0.0–1.0 floats.

| Role | RGBA | Usage |
|------|------|-------|
| Dominant surface (60%) | (0, 0, 0, 0) transparent + BasicFrameTemplateWithInset chrome | Config window background; pack frame background — WoW frame chrome is the dominant surface |
| Secondary surface (30%) | (1, 1, 1, 0.05) | Alternating pull row stripe (even rows) |
| Accent — selection (10%) | (0, 1, 0, 0.15) | Selected/ready pack row background |
| Active state | (1, 0.5, 0, 0.25) | Active/fighting pack row background (orange) |
| Completed state | (0.2, 0.2, 0.2, 0.3) | Completed pack rows (grey) |
| Boss pull | (0.6, 0.2, 0.2, 0.3) | Boss pull row tint (dark red) |
| Urgent / glow | (1, 0, 0, 1) | Red glow border on urgent icons (GLOW_WIDTH = 2px edges) |
| Hover highlight | (1, 1, 1, 0.1) | Row hover highlight texture (HIGHLIGHT layer) |
| Disabled | SetAlpha(0.4) on widget | TTS EditBox when sound is selected in dropdown |
| Header text — loaded | (1, 0.82, 0, 1) | Gold: dungeon name + pull count when route is loaded |
| Header text — empty | (0.6, 0.6, 0.6, 1) | Grey: "No route imported" |
| Body text | (1, 1, 1, 1) | White: icon labels, mob names, ability names |
| Dungeon header text | GameFontNormal default (white/gold) | Collapsible dungeon tree headers |

**Accent reserved for:** Selected pack row (green), active/fighting pack row (orange), urgent icon glow (red), loaded-route header text (gold). Never applied to generic labels or non-state elements.

**Destructive color:** No dedicated destructive color constant. Destructive actions use StaticPopup dialogs (WoW system dialogs handle their own styling). The "Reset" and "Reset All" buttons use standard GameMenuButtonTemplate with no special color — the confirmation dialog provides the destructive signal.

## Interaction Contracts

### Config Window Open/Close

- Opens via `/tpw config` slash command → `ns.ConfigUI.Toggle()`
- Optional: "Config" button in PackFrame footer → same call
- Escape key closes (UISpecialFrames registration)
- Position is not persisted in Phase 13 (defaults to CENTER; position persistence is a future improvement)
- Lazy construction: first `ns.ConfigUI.Toggle()` call builds all frames; subsequent calls just Show/Hide

### Left Panel — Dungeon Tree

- Dungeon headers are Buttons with text "▼ DungeonName" (expanded) or "► DungeonName" (collapsed)
- Click header → toggle `node.expanded`, call `RebuildLayout(leftScrollChild)`, update `leftScrollChild:SetHeight()`, call `leftScrollFrame:UpdateScrollChildRect()`
- Mob rows are Buttons: 22×22 NPC portrait (circular mask via TempPortraitAlphaMask) + mob name FontString
- Portrait fallback chain: `SetPortraitTextureFromCreatureDisplayID` → class icon → question mark (identical to PackFrame.lua `GetPortraitTexture`)
- Click mob row → load that mob's abilities in right panel (clear then rebuild right panel rows)
- Selected mob row: subtle highlight background, not a persistent selection color (right panel content is the signal)

### Right Panel — Mob Details

- Header FontString: "Mob Name — CLASS" (e.g., "Nerubian Spellguard — PALADIN")
- On mob selection: hide all existing `rightPanelRows`, rebuild from `ns.AbilityDB[npcID].abilities`
- Each skill row layout (top-to-bottom within row):
  1. Row 1: CheckButton (enabled) + spell icon (22×22, C_Spell.GetSpellTexture) + ability name label
  2. Row 2 (if timed): timing info FontString ("First cast: Xs, Cooldown: Xs") — hidden for untimed abilities
  3. Row 3: "Label:" prefix + labelEditBox | "Sound/TTS:" prefix + soundBtn | [Reset] button
  4. Row 4: "TTS Text:" prefix + ttsEditBox (full width minus padding)
- Skill rows separated by 4px vertical gap

### Per-Skill: Enabled Checkbox (CFG-03)

- CheckButton, checked by default
- State derived from: `ns.db.skillConfig[npcID][spellID].enabled == false` → unchecked; nil or true → checked
- OnClick: write `ns.db.skillConfig[npcID][spellID].enabled = false` (uncheck) or `= nil` (re-check, sparse schema — nil means enabled)
- Visual effect: disabled abilities take effect on next route import, not immediately

### Per-Skill: Label EditBox (CFG-04)

- Default text: `ability.label` from AbilityDB (e.g., "DR", "Kick")
- Empty string is valid (clears label from icon display)
- Write: `ns.db.skillConfig[npcID][spellID].label = editBox:GetText()` on confirm; set nil to restore default
- Focus handlers — BOTH required:
  ```lua
  editBox:SetScript("OnEnterPressed", function(self) self:ClearFocus() end)
  editBox:SetScript("OnEscapePressed", function(self) self:ClearFocus() end)
  ```

### Per-Skill: Sound Dropdown (ALERT-01, ALERT-03)

- Implemented as: Button showing current selection name + popup Frame with scroll list of options
- NOT UIDropDownMenu (deprecated in Midnight)
- Default selection text: "TTS" (soundKitID = nil in skillConfig means TTS mode)
- Sound options sourced from `ns.AlertSounds` (Data/Sounds.lua)
- On option click:
  - If soundKitID is non-nil: `PlaySound(soundKitID)` (preview), write `skillConfig.soundKitID = soundKitID`, disable TTS EditBox (`ttsEditBox:Disable()`, `ttsEditBox:SetAlpha(0.4)`)
  - If soundKitID is nil ("TTS"): clear `skillConfig.soundKitID`, enable TTS EditBox (`ttsEditBox:Enable()`, `ttsEditBox:SetAlpha(1.0)`)
- Sound popup: SetFrameStrata("DIALOG"), appears anchored below the sound button

### Per-Skill: TTS EditBox (ALERT-02, ALERT-03)

- Default text: `ability.ttsMessage` or `ability.name` if ttsMessage is nil
- Enabled state: only when sound dropdown shows "TTS"
- Disabled state: `EditBox:Disable()` + `SetAlpha(0.4)` when any non-TTS sound is selected
- Write: `ns.db.skillConfig[npcID][spellID].ttsMessage = editBox:GetText()` on confirm
- Same focus handlers as label EditBox (OnEnterPressed + OnEscapePressed both call ClearFocus)

### Per-Skill: Spell Tooltip (CFG-05)

- `skillRow:EnableMouse(true)`
- OnEnter: `GameTooltip_SetDefaultAnchor(GameTooltip, self)` → `GameTooltip:SetSpellByID(self.spellID)` → `GameTooltip:Show()`
- OnLeave: `GameTooltip:Hide()`
- Exact same pattern as IconDisplay.lua lines 89–96

### Per-Skill: Reset Button

- Text: "Reset"
- OnClick: `ns.db.skillConfig[npcID][spellID] = nil` (clears all overrides for this skill)
- Then refresh the right panel to show restored defaults

### Per-Dungeon: Reset All Button

- Text: "Reset All"
- OnClick: iterate all npcIDs for the selected dungeon (via DungeonEnemies[dungeonIdx] cross-referenced with AbilityDB), set `ns.db.skillConfig[npcID] = nil` for each
- No StaticPopup confirmation needed — reset is non-destructive (defaults are always available from AbilityDB)
- Refresh right panel after reset

### Mob Count Display — PackFrame Pull Rows (ROUTE-04)

- Count overlay is a FontString created lazily on `tex` (portrait texture object)
- Position: `SetPoint("BOTTOMRIGHT", tex, "BOTTOMRIGHT", 0, 0)`
- Font: `FRIZQT__.TTF`, 10px, OUTLINE
- Text color: white `(1, 1, 1, 1)`
- Logic: `if count > 1 then countLabel:SetText("x"..count); countLabel:Show() else countLabel:Hide() end`
- Count data: `pack.mobCounts[npcID]` populated by Pipeline.BuildPack before the seenNpc dedup guard
