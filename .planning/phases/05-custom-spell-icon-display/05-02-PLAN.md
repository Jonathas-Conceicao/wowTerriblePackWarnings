  <action>
Rewrite Engine/Scheduler.lua to replace all ns.BossWarnings calls with ns.IconDisplay calls. The Scheduler's structure (timer chain, combatActive guard, Start/Stop) stays the same; only the display integration changes.

**Changes to scheduleAbility(ability):**

1. **Remove the old pre-warn Show call** (line 40: `ns.BossWarnings.Show(ability.name .. " in 5 sec", 4)`).
   Replace with: `ns.IconDisplay.SetUrgent(barId)` — this adds the red glow border AND fires TTS automatically via the ttsMessage stored on the slot.

2. **Remove the old cast-alert Show call** (line 49: `ns.BossWarnings.Show(ability.name, 3)`).
   No replacement needed — the icon is already visible from ShowIcon. The cast event just triggers the next cycle.

3. **Replace ShowTimer call** (line 64: `ns.BossWarnings.ShowTimer(barId, ability.name, ability.first_cast, ability.spellID)`).
   Replace with: `ns.IconDisplay.ShowIcon(barId, ability.spellID, ability.ttsMessage, ability.first_cast)`
   Note: pass `ability.ttsMessage` (may be nil for abilities without TTS, which is fine — ShowIcon stores it on the slot for later SetUrgent use).

4. **In the cast callback's reschedule**: The recursive scheduleAbility call already creates a new barId via timerCounter increment. The ShowIcon call with a new barId will reuse the existing slot if the same instanceKey is used. To handle timer reset on repeating abilities, use the SAME barId pattern. Change the recursion to pass the current barId so ShowIcon resets the existing slot:

   The recursion should pass the current barId to allow icon reuse:
   ```lua
   -- Change scheduleAbility signature to accept optional existingBarId
   scheduleAbility = function(ability, existingBarId)
       local barId = existingBarId or ("bar_" .. timerCounter)
       if not existingBarId then
           timerCounter = timerCounter + 1
       end
       -- ... pre-warn uses barId for SetUrgent ...
       -- ... cast callback calls scheduleAbility(nextAbility, barId) to reuse slot ...
   ```

   In the cast callback, pass barId to the recursive call so the next cycle reuses the same icon slot (ShowIcon resets the cooldown sweep when the key already exists).

**Changes to Scheduler:Start(dungeonKey, packIndex):**

5. **Add untimed ability handling**: The current Start loop skips untimed abilities (`if ability.cooldown then`). Add an else branch:
   ```lua
   for _, ability in ipairs(pack.abilities) do
       if ability.cooldown then
           scheduleAbility(ability)
       else
           -- Untimed: show static icon (one per ability, regardless of mob count)
           ns.IconDisplay.ShowStaticIcon("static_" .. ability.spellID, ability.spellID)
           dbg("Static icon: " .. ability.name)
       end
   end
   ```

**Changes to Scheduler:Stop():**

6. **Replace CancelAllTimers** (line 109: `pcall(ns.BossWarnings.CancelAllTimers)`).
   Replace with: `pcall(ns.IconDisplay.CancelAll)` — same pcall protection pattern.
