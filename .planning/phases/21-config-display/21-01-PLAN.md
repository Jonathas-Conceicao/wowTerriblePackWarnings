1. In ApplySearchFilter, change line 1142 from:
   ```lua
   local filter = text:lower()
   ```
   to:
   ```lua
   local filter = text:lower():gsub("%-", "")
   ```
   This strips hyphens from user input so "mini-boss" becomes "miniboss" and matches naturally. Note: gsub does NOT have a plain-text flag, so the hyphen MUST be escaped as "%-" in the Lua pattern.

2. After the mobNameMatch check (line 1152) and before the else branch (line 1158), add a categoryMatch check. The new structure for lines 1152-1176 should be:
   ```lua
   local mobNameMatch = (mob.name or ""):lower():find(filter, 1, true)

   -- Category match: check mob's category against search filter
   local categoryMatch = false
   local entry = ns.AbilityDB[npcID]
   if entry and entry.mobCategory then
       categoryMatch = entry.mobCategory:find(filter, 1, true) ~= nil
   end

   if mobNameMatch or categoryMatch then
       -- Mob name or category match: show all abilities
       currentMatchedMobs[npcID] = true
       -- Do NOT populate currentMatchedSpells for this mob (show all)
   else
       -- Check if any ability name matches (existing code, unchanged)
       local entry = ns.AbilityDB[npcID]
       ...
   end
   ```

3. CRITICAL: categoryMatch must be in the SAME branch as mobNameMatch (both set currentMatchedMobs without currentMatchedSpells). Do NOT put category matching inside the else block — that would filter to specific abilities instead of showing all abilities for matched mobs.

4. Note: the `entry` local in the category check and the `entry` local in the else block are in different scopes, so no conflict. But if you prefer, rename the category one to `catEntry` for clarity.
