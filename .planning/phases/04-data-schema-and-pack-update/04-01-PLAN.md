**Engine/Scheduler.lua — three changes:**

1. **Line 85** — Update debug message from `#pack.mobs .. " mobs"` to `#pack.abilities .. " abilities"`:
   ```lua
   dbg("Start: " .. pack.displayName .. " (" .. #pack.abilities .. " abilities)")
   ```

2. **Lines 87-91** — Replace the nested mob/ability loop with a flat loop over pack.abilities, adding a nil-cooldown guard to skip untimed abilities:
   ```lua
   for _, ability in ipairs(pack.abilities) do
       if ability.cooldown then
           scheduleAbility(ability)
       else
           dbg("Skip untimed: " .. ability.name)
       end
   end
   ```
   The `if ability.cooldown then` guard is critical — scheduleAbility reads `ability.first_cast - 5` on line 31 and `ability.cooldown` on line 48, both of which would error on nil. Untimed abilities (Spirit Bolt) must be skipped by the Scheduler entirely. Phase 5 will handle displaying them as static icons.

**Engine/CombatWatcher.lua — no changes needed.** CombatWatcher does not iterate pack.mobs directly; it only calls Scheduler:Start which handles the iteration. The references to `pack.displayName` and `dungeon[packIndex].displayName` remain valid since `displayName` is kept on pack entries.
