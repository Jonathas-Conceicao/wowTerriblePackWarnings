<task type="auto">
  <name>Task 2: Create Import/Pipeline.lua with full import pipeline</name>
  <files>Import/Pipeline.lua</files>
  <action>
Create Import/Pipeline.lua implementing the ns.Import module with four public functions: RunFromPreset, RunFromString, RestoreFromSaved, Clear.

Follow the research pattern from 09-RESEARCH.md Pattern 1 exactly. Key implementation details:

1. **DUNGEON_IDX_MAP** (local constant): Maps all 9 Midnight dungeonIdx values to { key, name } pairs. Used for display name resolution and identifying unknown dungeons.

2. **BuildPack(pullIdx, pullData, dungeonIdx)** (local function):
   - Creates a pack with displayName = "Pull " .. pullIdx
   - Looks up ns.DungeonEnemies[dungeonIdx] for enemy data
   - If enemies table is nil (unknown dungeon), returns pack with empty npcIDs/abilities
   - Iterates pullData with `pairs()`, guards with `if tonumber(enemyIdx) then` to skip "color" key
   - Also guards `enemies[enemyIdx]` exists before accessing .id
   - Deduplicates npcIDs within the pull using seenNpc table
   - Deduplicates abilities using seenAbility table keyed by spellID_mobClass
   - For each unique npcID: looks up ns.AbilityDB[npcID], copies abilities into pack
   - Shallow-copies each ability field (name, spellID, mobClass, first_cast, cooldown, label, ttsMessage)

3. **Import.RunFromPreset(preset)**:
   - Validates preset.value.currentDungeonIdx and preset.value.pulls exist
   - Resolves dungeon name from DUNGEON_IDX_MAP (fallback: "Dungeon #N" for unknown)
   - Iterates pulls[1..#pulls] sequentially (preserves order)
   - Calls BuildPack for each pull
   - Assigns ns.PackDatabase["imported"] = packs
   - Saves to ns.db.importedRoute = { dungeonName, dungeonIdx, packs }
   - Prints summary: "Imported: Windrunner Spire - 17 pulls (4 with tracked abilities)"
   - Calls ns.CombatWatcher:SelectDungeon("imported")
   - Calls ns.PackUI:Refresh() if available
   - Returns true on success

4. **Import.RunFromString(importString)**:
   - Calls ns.MDTDecode(importString)
   - On failure: prints error, returns false
   - On success: calls Import.RunFromPreset(result)

5. **Import.RestoreFromSaved()**:
   - If ns.db.importedRoute is nil, returns immediately
   - Reads saved.packs, assigns to ns.PackDatabase["imported"]
   - Prints restore summary
   - Calls SelectDungeon and UI refresh

6. **Import.Clear()**:
   - Sets ns.PackDatabase["imported"] = nil
   - Sets ns.db.importedRoute = nil
   - Stops NameplateScanner and Scheduler
   - Prints confirmation
   - Refreshes UI

CRITICAL anti-patterns to avoid:
- Do NOT use `pairs()` to iterate pulls array — use `for pullIdx = 1, #pulls` to preserve order
- DO use `tonumber(enemyIdx)` guard when iterating pullData entries
- Do NOT save raw MDT string — save processed pack data only
- Do NOT add "imported" to ZONE_DUNGEON_MAP — zone changes should clear state
