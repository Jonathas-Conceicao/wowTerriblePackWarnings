<task type="auto">
  <name>Task 2: Wire selection click handlers and combat state row rendering</name>
  <files>UI/PackFrame.lua</files>
  <action>
**Update ElementInitializer for pack leaf rows (OnClick):**
- Replace the no-op placeholder OnClick with: `ns.CombatWatcher:SelectPack(data.dungeonKey, data.packIndex)`
- This sets the dungeon + pack index and triggers Refresh via the callback added in Task 1

**Add ns.PackUI:Refresh() function:**
- Read current state: `local state, activeDungeon, activePackIndex = ns.CombatWatcher:GetState()`
- Trigger ScrollBox to re-render all visible elements by calling `ScrollBox:ForEachFrame(function(button) ... end)` or by refreshing the DataProvider
- The simplest approach for a small list: rebuild DataProvider (call PopulateList again which creates a new TreeDataProvider and calls SetDataProvider). This avoids tracking individual button references.
- After rebuilding, apply state-based styling to each row

**Add UpdateRowAppearance logic inside the ElementInitializer:**
- After setting text for pack leaf rows, call a local `UpdateRowAppearance(button, data)` function
- This function reads `ns.CombatWatcher:GetState()` each time a row initializes (ScrollBox re-initializes rows on data provider change)
- State-based styling for pack rows:
  - **Selected/ready** (state == "ready" and dungeon+pack match): green text color `(0, 1, 0)` + prepend checkmark icon string `"|TInterface\\Buttons\\UI-CheckBox-Check:16|t "` before pack name
  - **Active/fighting** (state == "active" and dungeon+pack match): orange text color `(1, 0.5, 0)` + prepend swords icon `"|TInterface\\LFGFrame\\BattlenetWorking0:16|t "` or simply use `"|A:common-icon-redx:16:16|a "` -- choose a visible combat indicator. Claude's discretion on exact icon.
  - **Completed** (same dungeon, packIndex < activePackIndex): grey text `(0.5, 0.5, 0.5)` + prepend checkmark `"|TInterface\\Buttons\\UI-CheckBox-Check:16|t "`
  - **Default** (not selected, not completed): white text `(1, 1, 1)`, no icon prefix

**Dungeon header styling in ElementInitializer:**
- Dungeon headers that contain the currently selected dungeon: slightly highlighted text (e.g., `(1, 0.82, 0)` gold)
- Other dungeon headers: normal white text

**Implement Refresh as PopulateList wrapper:**
- `function ns.PackUI:Refresh() PopulateList() end`
- PopulateList rebuilds the DataProvider from scratch, which causes ScrollBox to re-initialize all elements via ElementInitializer, which reads current state for styling
- This is safe for a small pack list (one dungeon with a few packs). For v1 this is sufficient.

**Ensure clicking completed packs works for wipe recovery:**
- The OnClick handler calls `CombatWatcher:SelectPack()` regardless of current state -- this allows re-selecting any pack including completed ones
