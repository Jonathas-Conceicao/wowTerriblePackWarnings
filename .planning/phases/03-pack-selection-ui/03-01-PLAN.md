<task type="auto">
  <name>Task 1: Create PackFrame.lua window with accordion ScrollBox tree list</name>
  <files>UI/PackFrame.lua</files>
  <action>
Create `UI/PackFrame.lua` with the standard `local addonName, ns = ...` namespace pattern.

**Frame setup:**
- Create main frame using `BasicFrameTemplateWithInset` template (provides title bar + close button): `CreateFrame("Frame", "TPWPackFrame", UIParent, "BasicFrameTemplateWithInset")`
- Size: 300x400, default position CENTER
- Set `frame.TitleText:SetText("TerriblePackWarnings")`
- Make movable: `SetMovable(true)`, `EnableMouse(true)`, `RegisterForDrag("LeftButton")`, wire `OnDragStart` to `StartMoving`, wire `OnDragStop` to `StopMovingOrSizing` + save position to `ns.db.windowPos = { x = self:GetLeft(), y = self:GetTop() }`
- Register with `UISpecialFrames`: `tinsert(UISpecialFrames, "TPWPackFrame")` (Escape to close)
- Start hidden: `frame:Hide()`

**Position restore:**
- Create `RestorePosition()` function that checks `ns.db and ns.db.windowPos`, then calls `frame:ClearAllPoints()` and `frame:SetPoint("TOPLEFT", UIParent, "BOTTOMLEFT", pos.x, pos.y)` (BOTTOMLEFT anchor per research pitfall #1)
- Register for `ADDON_LOADED` on an internal event frame to call `RestorePosition()` after `ns.db` is available. Since Core.lua handles ADDON_LOADED first and sets ns.db, the PackFrame ADDON_LOADED handler fires in the same event pass. Alternatively, use a small delay or check `ns.db` existence.

**ScrollBox tree list (accordion):**
- Create ScrollBox: `CreateFrame("Frame", nil, frame.Inset, "WowScrollBoxList")`
- Anchor to fill `frame.Inset` with 4px padding on all sides
- Create ScrollBar: `CreateFrame("EventFrame", nil, frame.Inset, "MinimalScrollBar")`, anchor to right of ScrollBox
- Create ScrollView: `CreateScrollBoxListTreeListView()`
- Wire: `ScrollUtil.InitScrollBoxListWithScrollBar(ScrollBox, ScrollBar, ScrollView)`

**Element initializer:**
- Use `"UIPanelButtonTemplate"` for row buttons
- In the initializer function, check `node:GetData().isDungeon`:
  - If dungeon header: set text to display name (use a local `DUNGEON_NAMES` lookup table mapping keys like `"windrunner_spire"` to `"Windrunner Spire"`), set OnClick to `node:ToggleCollapsed()` then call `ScrollBox:Update()` (forces re-layout). Style header text with `GameFontNormal` color or slightly brighter to distinguish from pack rows.
  - If pack leaf: set text to `data.displayName`, set OnClick as a no-op placeholder for now (Plan 02 wires selection logic)
- Call `ScrollView:SetElementInitializer("UIPanelButtonTemplate", ElementInitializer)`

**Data population:**
- Create `PopulateList()` function that:
  1. Creates a `CreateTreeDataProvider()`
  2. Iterates `ns.PackDatabase` with `pairs()`, for each `dungeonKey, packs`:
     - Insert dungeon node: `{ isDungeon = true, displayName = DUNGEON_NAMES[dungeonKey] or dungeonKey, key = dungeonKey }`
     - For each pack `i, pack` in `ipairs(packs)`: insert child node: `{ isDungeon = false, displayName = pack.displayName, dungeonKey = dungeonKey, packIndex = i }`
  3. Calls `ScrollView:SetDataProvider(DataProvider)`
- Call `PopulateList()` at the end of the file (data files load before UI in TOC)

**Public API (ns.PackUI):**
- `ns.PackUI = {}` at top of file
- `ns.PackUI.Toggle = function() if frame:IsShown() then frame:Hide() else frame:Show() end end`
- `ns.PackUI.Show = function() frame:Show() end`
- `ns.PackUI.Hide = function() frame:Hide() end`
- Store references to ScrollBox, ScrollView, DataProvider on the module table for Plan 02 to use in Refresh

**Dungeon display names:**
- Add a local lookup table at the top: `local DUNGEON_NAMES = { windrunner_spire = "Windrunner Spire" }`
- This addresses Research open question #2 without modifying PackDatabase schema

**Core.lua — Restore window position after ADDON_LOADED:**
- After `ns.db = TerriblePackWarningsDB` line in the ADDON_LOADED handler, add: `if ns.PackUI and ns.PackUI.RestorePosition then ns.PackUI.RestorePosition() end`
- This calls PackFrame's position restore after SavedVariables are loaded. The nil guard is safe because Core.lua loads before PackFrame.lua.
- NOTE: Actually, Core.lua ADDON_LOADED fires once and unregisters. PackFrame.lua loads after Core.lua in TOC, so ns.PackUI won't exist when ADDON_LOADED fires in Core.lua. The safest approach: in PackFrame.lua's PopulateList or at end of file, call RestorePosition() directly since ns.db is already set by the time PackFrame.lua executes. Update Task 1's RestorePosition to be called at end of file, not in an event handler.
