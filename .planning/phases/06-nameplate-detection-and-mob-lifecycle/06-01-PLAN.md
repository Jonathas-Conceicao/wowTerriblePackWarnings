  <name>Task 1: Refactor Scheduler with StartAbility/StopAbility and per-barId timer tracking</name>
  <files>Engine/Scheduler.lua</files>
  <action>
Modify Engine/Scheduler.lua to add per-mob timer control while keeping the existing Scheduler:Start and Scheduler:Stop working (they will be replaced by scanner calls in Plan 02, but must not break during this plan).

1. Add a `barTimers` table at module scope: `local barTimers = {}` — maps barId to `{ handles = {} }` where handles is a list of C_Timer handles for that barId.

2. Inside `scheduleAbility`, after each `C_Timer.NewTimer` call that creates `preHandle` or `castHandle`, in addition to `table.insert(activeTimers, handle)`, also track per-barId:
   ```lua
   barTimers[barId] = barTimers[barId] or { handles = {} }
   table.insert(barTimers[barId].handles, handle)
   ```
   Do this for both the pre-warn timer and the cast timer.

3. Add public `Scheduler:StartAbility(ability, barId)`:
   ```lua
   function Scheduler:StartAbility(ability, barId)
       combatActive[1] = true
       scheduleAbility(ability, barId)
   end
   ```
   This is the entry point the scanner will call per-mob. It ensures combatActive is set (scanner may call before old Start) and delegates to existing scheduleAbility with an explicit barId.

4. Add public `Scheduler:StopAbility(barId)`:
   ```lua
   function Scheduler:StopAbility(barId)
       local entry = barTimers[barId]
       if entry then
           for _, handle in ipairs(entry.handles) do
               if not handle:IsCancelled() then
                   handle:Cancel()
               end
           end
           barTimers[barId] = nil
       end
       ns.IconDisplay.CancelIcon(barId)
   end
   ```
   This cancels only the timers for one mob's barId and removes its icon.

5. In `Scheduler:Stop()`, add `wipe(barTimers)` after the existing `wipe(activeTimers)` line to clean up per-barId tracking on full combat end.

Do NOT remove or change the existing `Scheduler:Start(dungeonKey, packIndex)` or `Scheduler:Stop()` signatures — they still work and Plan 02 will change the callers.
  </action>

  <done>Scheduler.lua has StartAbility(ability, barId) and StopAbility(barId) public methods, per-barId timer tracking in barTimers table, and Stop() wipes barTimers. Existing Start/Stop still functional.</done>

<task type="auto">
  <name>Task 2: Create NameplateScanner module with poll-based mob detection and lifecycle</name>
  <files>Engine/NameplateScanner.lua</files>
  <action>
Create new file `Engine/NameplateScanner.lua` implementing the 0.25s nameplate polling scanner. Use `local addonName, ns = ...` namespace pattern.

Module structure:

1. **Namespace and state:**
   ```lua
   ns.NameplateScanner = {}
   local Scanner = ns.NameplateScanner

   local activePack = nil        -- current pack reference (has .abilities)
   local tickerHandle = nil      -- C_Timer.NewTicker handle
   local prevCounts = {}         -- classBase -> number (count from last tick)
   local classBarIds = {}        -- classBase -> { barId1, barId2, ... }
   local staticShown = {}        -- spellID -> boolean (untimed icon shown)
   local timerCounter = 0        -- global incrementing ID for unique barIds
   ```

2. **Scanner:Start(pack):**
   - Guard: `if tickerHandle then return end` (prevent duplicate tickers per Pitfall 3)
   - Set `activePack = pack`
   - Wipe prevCounts, classBarIds, staticShown, reset timerCounter to 0
   - Create ticker: `tickerHandle = C_Timer.NewTicker(0.25, function() Scanner:Tick() end)`
   - Do an immediate first tick: `Scanner:Tick()` (don't wait 0.25s for first detection)

3. **Scanner:Stop():**
   - Cancel ticker if exists and not cancelled
   - Set tickerHandle = nil, activePack = nil
   - `wipe(prevCounts); wipe(classBarIds); wipe(staticShown)`

4. **Scanner:Tick():**
   - Guard: `if not activePack then return end`
   - Build `newCounts = {}` by iterating `C_NamePlate.GetNamePlates()`
   - For each plate: `local npUnit = plate.namePlateUnitToken or plate.unitToken`
   - Guard: `if not npUnit then skip end`
   - Check hostile: `UnitCanAttack("player", npUnit)` (no pcall needed for this one, it's safe)
   - Check in-combat: `local inCombatOk, inCombat = pcall(UnitAffectingCombat, npUnit)` — skip if not ok or not inCombat
   - Get class: `local classOk, _, classBase = pcall(UnitClass, npUnit)` — skip if not ok or not classBase
   - Increment: `newCounts[classBase] = (newCounts[classBase] or 0) + 1`
   - After iteration, reconcile increases then decreases (order matters), then set `prevCounts = newCounts`

5. **Scanner:OnMobsAdded(classBase, delta):**
   - Initialize `classBarIds[classBase] = classBarIds[classBase] or {}`
   - For each ability in `activePack.abilities` where `ability.mobClass == classBase`:
     - If `ability.cooldown` (timed): loop `delta` times, increment timerCounter, generate barId as `"mob_" .. classBase .. "_" .. timerCounter`, insert into classBarIds[classBase], call `ns.Scheduler:StartAbility(ability, barId)`
     - If not `ability.cooldown` (untimed): if `not staticShown[ability.spellID]`, set it true and call `ns.IconDisplay.ShowStaticIcon("static_" .. ability.spellID, ability.spellID)`

6. **Scanner:OnMobsRemoved(classBase, delta):**
   - Get `ids = classBarIds[classBase]`, return if nil
   - Loop `delta` times: `local barId = table.remove(ids)` (remove from end), if barId then `ns.Scheduler:StopAbility(barId)`
   - After removals, if `#ids == 0`: clear static icons for this class by iterating activePack.abilities where mobClass matches and cooldown is nil, calling `ns.IconDisplay.CancelIcon("static_" .. ability.spellID)` and setting `staticShown[ability.spellID] = nil`. Then set `classBarIds[classBase] = nil`.

7. **Reconcile logic in Tick (increases):**
   ```lua
   for classBase, count in pairs(newCounts) do
       local prev = prevCounts[classBase] or 0
       if count > prev then
           Scanner:OnMobsAdded(classBase, count - prev)
       end
   end
   ```

8. **Reconcile logic in Tick (decreases):**
   ```lua
   for classBase, prev in pairs(prevCounts) do
       local count = newCounts[classBase] or 0
       if count < prev then
           Scanner:OnMobsRemoved(classBase, prev - count)
       end
   end
   ```

Include debug logging using the established `|cff888888TPW-dbg|r` prefix pattern for Start, Stop, Tick counts, OnMobsAdded, and OnMobsRemoved.
  </action>
